# AbDiscovery OS — AI驱动抗体研发数据平台 · 设计说明

> 最后更新：2026-06-17
> 面试目标：百图生科 — 生物研发平台产品经理（抗体/细胞/合成生物学）

---

## 一、原型定位

AbDiscovery OS 是一个 **AI驱动的抗体研发数据平台原型**，以抗体发现全流程为业务场景，产品架构对标 **Benchling + BioMap OS** 双平台。

核心叙事：

> 我不仅能做一个"AI功能页"，而是能把 AI 放进一个真实的抗体研发流程里，围绕实体管理、实验数据、智能分析和干湿闭环去构建产品。

原型覆盖抗体研发 8 个核心阶段：

| 阶段 | 英文 | 说明 |
|------|------|------|
| 1. 靶点发现与验证 | Target Discovery | 靶点筛选、验证、可药性评估 |
| 2. 抗体库构建与筛选 | Library & Screening | 噬菌体展示/AI生成，高通量筛选 |
| 3. 命中确认 | Hit Confirmation | ELISA验证、初步亲和力测定 |
| 4. 先导优化 | Lead Optimization | 亲和力成熟、人源化、工程化改造 |
| 5. 可开发性评估 | Developability | 稳定性、聚集、粘度、免疫原性 |
| 6. 细胞株构建 | Cell Line Dev | CHO/HEK转染、高表达克隆筛选 |
| 7. 工艺开发 | Process Dev | 上游培养、下游纯化工艺 |
| 8. 候选分子确定 | Candidate Selection | 综合评估，进入IND申报 |

## 二、设计原则

### 2.1 对标双平台

每个页面的侧边栏标注两层对标关系：
- **Benchling 对标**：该页面对应 Benchling 的哪个产品模块
- **BioMap OS 映射**：该页面对应百图生科 BioMap OS 的哪个模块

### 2.2 科学家优先

系统适应科学家的工作方式，而非反过来。结构化数据采集嵌入实验记录流程，对科学家零额外负担。数据录入表单直接内嵌在实验数据页面，而非独立入口。

### 2.3 干湿闭环

核心叙事不是"AI帮你做一件事"，而是**门控闭环**（Gated Loop）：

```
AI序列设计 → 湿实验验证 → 数据回传 → AI分析 → 下一轮设计
```

4 轮迭代从 KD=12.5nM 优化到 KD=0.3nM，展示 AI 与实验室人机协同的完整链条。

### 2.4 AI不是功能，是交互方式

AI Agent 嵌入科学家自然工作流：对话式查询、自动解析实验报告、智能推荐、主动式分析，而非一个单独的"AI 聊天入口"。

### 2.5 全链路溯源

从靶点 → 抗体序列 → 质粒 → 细胞株 → 实验数据，每个实体有唯一 ID，关联关系可视化展示，实现 Benchling Registry 式的全链路追踪。

## 五、6 个一级页面说明

### 页面 1：靶点研究

**对标**: Benchling Deep Research Agent / BioMap xTrimo发现助手

页面内容：
- 靶点选择下拉框（PD-L1、HER2 两个靶点可切换）
- 靶点基本信息卡片（基因名、UniProt ID、蛋白家族、PDB结构、分子量、组织表达、作用机制）
- 已上市/在研竞品看板 + 年销售额柱状图
- AI可药性评估雷达图（4 维：表面可及性、构象稳定性、已知表位、安全性信号）
- 文献情报摘要（模拟 RAG 检索结果）
- AI立项建议（评分 + 风险 + 差异化策略 + 下一步）

体现的产品思考：
- 靶点研究是抗体研发的"0号阶段"，PM 需要理解科学家如何做立项决策
- AI 不只是检索工具，而是能给出**可操作建议**（谨慎推进 vs 积极推进）

### 页面 2：序列注册中心（JD核心页面）

**对标**: Benchling Registry + Inventory / BioMap 样本全流程管理

这是直接回应 JD 第一条核心职责「样本全流程管理」的关键页面。

页面内容：
- KPI 指标（15条抗体、6个细胞株、8个质粒、AI生成序列数）
- **实体 Schema 定义**（3 类核心实体的数据结构）：
  - 抗体序列 Schema：13 个字段（ID/靶点/格式/VH/VL/CDR/KD/Tm/来源/项目...）
  - 细胞株 Schema：11 个字段（ID/宿主/质粒/传代/滴度/活力/位置...）
  - 质粒 Schema：8 个字段（ID/骨架/插入基因/抗性/启动子...）
- **注册表浏览器**：Tab 切换三类实体，可按靶点/格式/来源筛选
  - 抗体序列带亲和力颜色编码（KD<1nM 绿 / 1-10nM 黄 / >10nM 红）
  - 展开查看序列详情，可跳转到抗体详情二级页面
- **实体关系网络**：Sankey 图展示 靶点→抗体→质粒→细胞株 全链路
- **库存状态面板**：样本存储位置、状态（生产株/瞬转株）

体现的产品思考：
- 用 Schema 定义展示「线下流程→标准化产品功能」的产品经理方法论
- 实体关系网络体现全链路溯源能力
- "科学家录入数据"和"企业获取结构化数据"不是对立的

### 页面 3：AI序列设计

**对标**: Benchling Model Hub + Biologics / BioMap xTrimo/AIGP

直接映射百图生科的 AIGP（AI Generated Protein）平台三种模式：

| 模式 | 含义 | 应用场景 |
|------|------|---------|
| F2P (Function→Protein) | 从功能需求生成序列 | 输入靶点+期望KD+格式，AI从零生成 |
| P2P (Protein→Protein) | 从已有序列优化 | 输入已有序列，AI做亲和力/稳定性优化 |
| C2P (Context→Protein) | 从上下文设计序列 | 输入表位信息+竞品序列，AI差异化设计 |

页面内容：
- 3 个 Tab 对应 3 种设计模式，每个 Tab 有输入表单和模拟生成结果
- AI 生成结果展示：候选序列列表（CDR序列、预测KD、预测Tm、可开发性评分、免疫原性风险）
- 模型信息卡片（对标 xTrimo：2100亿参数、训练数据、验证指标）

体现的产品思考：
- 直接使用百图生科的产品术语（AIGP / F2P / P2P / C2P），展示对公司产品的深度研究
- AI 序列设计不是黑盒，产品经理需要理解输入/输出/置信度

### 页面 4：实验数据管理（JD核心页面）

**对标**: Benchling Notebook + Structured Tables / BioMap 实验数据记录模块

页面内容：
- KPI 指标（ELISA 8条、SPR 8条、表达 6条、最优KD）
- **3 类 Structured Table**：
  - ELISA 结合活性表：抗体ID / OD450 / EC50 / 最大结合率 / 判定
  - SPR 动力学表：抗体ID / ka / kd / KD / Chi² / 判定
  - 细胞表达表：细胞株ID / 培养天数 / 滴度 / 活力 / 判定
- 异常预警（Chi²>0.4 / 活力<92%）
- ELISA 剂量-响应曲线（可选择多抗体对比）
- SPR 亲和力演进趋势图（传统筛选 vs AI设计的KD对比）
- 多候选抗体雷达图对比（亲和力/稳定性/表达/来源/综合 5维）
- **新增实验数据录入表单**（3类实验类型可选，提交后模拟写入）

体现的产品思考：
- Structured Table 不是简单的电子表格，而是有 Schema 约束的结构化数据采集
- 异常预警体现"系统主动服务科学家"的理念
- 数据录入表单体现"零切换成本"——科学家在数据管理页面内直接录入

### 页面 5：智能分析与闭环

**对标**: Benchling Insights + Automation / BioMap 智能分析 + 干湿闭环

页面内容：
- **SPR Sensorgram 可视化**：模拟结合-解离曲线，多候选叠加对比
- **可开发性评估看板**：
  - 5 维评估矩阵（Tm / 聚集% / 粘度 / 免疫原性 / 化学稳定性）
  - 通过/条件通过/不通过分类 + AI建议
  - 雷达图可视化
- **干湿闭环可视化**（核心展示）：
  - Plotly 流程图：AI设计→ELISA筛选→SPR验证→AI分析→下一轮（带箭头和Loop Back回路）
  - 4 轮迭代详情时间线
  - 迭代进展双轴图（序列数量 + KD演进）
- **效率对比**：传统12个月→AI辅助4个月，筛选量提升100倍

4 轮闭环迭代数据：

| 轮次 | AI动作 | 湿实验 | 结果 | 最优KD |
|------|--------|--------|------|--------|
| Round 1 | F2P生成100条候选 | ELISA初筛 | 8个Hit | 12.5 nM |
| Round 2 | P2P优化80条变体 | SPR验证 | 3个Lead (KD<5nM) | 0.8 nM |
| Round 3 | P2P精细成熟30条 | SPR+DSF+SEC三重验证 | 1个Lead确认 | 0.3 nM |
| Round 4 | C2P可开发性工程化 | 全面可开发性评估 | 最终候选AB-005确定 | 0.3 nM |

体现的产品思考：
- 干湿闭环是百图生科的核心差异化叙事，用4轮数据具体化展示
- 不是"AI替代科学家"，而是"AI+科学家在门控闭环中协同"

### 页面 6：AI知识助手

**对标**: Benchling AI Agents (Ask/Research/Design/Data Entry) / BioMap xTrimo知识助手

页面内容：
- **AI Agent 矩阵**（对标 Benchling 4 类 Agent）：

| Agent | 功能 | BioMap对标 |
|-------|------|-----------|
| Ask Agent | 对话式查询实验数据 | 知识助手 |
| Research Agent | 跨数据集深度分析 | 发现助手 |
| Design Agent | AI驱动序列设计 | AIGP |
| Data Entry Agent | 自动解析实验报告 | 数据整合 |

- **RAG 架构说明**：知识库(UniProt/PDB/文献/内部数据)→向量检索→大模型生成→引用溯源
- **5 组预设问答**（均为抗体工程专业问题，带学术文献来源）：
  1. IgG4与IgG1的区别和选择策略
  2. 如何提高抗体热稳定性（Tm）
  3. 双特异性抗体的主要构型和优劣
  4. CDR区域哪些突变最影响亲和力
  5. CHO细胞表达系统常见问题和优化
- **自定义问题输入** + **多轮对话演示**

体现的产品思考：
- AI Agent 不是单一聊天窗口，而是按场景分化的多Agent体系
- RAG架构展示对AI产品技术栈的理解
- 5组问答的深度体现对抗体工程领域的专业积累

## 十二、面试汇报要点

### 开场（30秒）

> "我做了一个 AI 驱动的抗体研发数据平台原型，叫 AbDiscovery OS。它不是一个散页式的 AI 功能演示，而是围绕抗体发现全流程——从靶点发现到候选分子确定——去构建产品架构。产品设计对标 Benchling 和 BioMap OS 双平台。"

### 核心卖点（按JD逐条回应）

1. **样本全流程管理**
   → 展示序列注册中心页面：3类实体Schema + Sankey关系图 + 库存面板

2. **实验数据记录**
   → 展示实验数据管理页面：3类Structured Tables + 异常预警 + 数据录入表单

3. **智能分析 + AI赋能**
   → 展示智能分析与闭环页面：SPR sensorgram + 可开发性评估 + 4轮干湿闭环
   → 展示AI序列设计页面：F2P/P2P/C2P 三模式，直接映射AIGP

4. **项目追踪**
   → 首页管线 + 项目详情二级页面钻取

5. **拆解复杂业务→标准化产品功能**
   → 用Schema定义展示方法论：线下实验记录→标准化字段→结构化数据→AI可分析

### 差异化表达

> "这个原型展示的不只是我能做产品，而是我理解抗体研发的业务逻辑。我知道 KD 是 kd/ka，知道 CHO 和 HEK293 的表达差异，知道可开发性评估要看 Tm、聚集、粘度、免疫原性。我也研究了百图生科的 xTrimo 和 AIGP 平台，所以在 AI 序列设计页面直接使用了 F2P/P2P/C2P 的产品术语。"

### 产品设计理念总结

> "三个核心理念：第一，科学家优先——系统适应工作方式，不增加额外负担；第二，干湿闭环——AI 和实验室不是两个世界，而是在门控闭环中持续迭代；第三，AI 不是功能，是交互方式——Agent 嵌入自然工作流，而不是一个单独的聊天入口。"
