# AbDiscovery OS

AI 驱动的抗体研发数据平台原型，覆盖从靶点发现到候选分子确定的关键流程。项目以抗体发现为业务主线，产品架构对标 `Benchling + BioMap OS`。

## 项目定位

AbDiscovery OS 不是实验室生产系统，而是一套用于产品设计展示、业务理解表达和方案沟通的交互原型。

它重点回答三个问题：

1. 抗体研发全流程能否被拆解成标准化的平台模块。
2. AI 能否嵌入研发链路，而不是停留在孤立功能页。
3. 面向抗体研发的平台产品，如何同时承接实体管理、实验数据、智能分析和项目推进。

## 核心叙事

项目围绕抗体研发 8 个核心阶段组织：

1. 靶点发现与验证
2. 抗体库构建与筛选
3. 命中确认
4. 先导优化
5. 可开发性评估
6. 细胞株构建
7. 工艺开发
8. 候选分子确定

平台中的页面不是孤立页面集合，而是围绕同一批项目、抗体、细胞株、质粒和实验数据构成一个连通原型。

## 功能模块

当前版本包含 6 个一级模块和 3 个二级钻取页面：

| 模块 | 说明 | 对标方向 |
|------|------|----------|
| 靶点研究 | 靶点基础信息、竞品情报、AI 可药性评估、立项建议 | Benchling Research Agent / BioMap 发现助手 |
| 序列注册中心 | 抗体、细胞株、质粒 Registry，Schema 定义与实体关系追踪 | Benchling Registry / BioMap 样本管理 |
| AI 序列设计 | AIGP 三模式：F2P / P2P / C2P | Benchling Model Hub / BioMap xTrimo |
| 实验数据管理 | ELISA、SPR、表达数据的结构化采集与展示 | Benchling Notebook / BioMap 实验记录 |
| 智能分析与闭环 | SPR 分析、可开发性评估、干湿闭环迭代 | Benchling Insights / BioMap 干湿闭环 |
| AI 知识助手 | Agent 矩阵、RAG 问答、多轮交互 | Benchling AI Agents / BioMap 知识助手 |
| 项目详情 | 项目维度的进度、候选与实验汇总 | 二级页面 |
| 抗体详情 | 抗体实体的序列、亲和力、可开发性和关联关系 | 二级页面 |
| 实验报告 | 项目级实验总结与 AI 结论输出 | 二级页面 |

## 设计亮点

### 1. 科学家优先

产品强调结构化数据采集嵌入研发流程，而不是要求科学家额外适应系统。

### 2. 干湿闭环

核心链路是：

`AI 序列设计 → 湿实验验证 → 数据回传 → AI 分析 → 下一轮设计`

项目中用 4 轮闭环迭代模拟了从 `KD = 12.5 nM` 优化到 `KD = 0.3 nM` 的过程。

### 3. 全链路溯源

从靶点到抗体，再到质粒、细胞株和实验数据，关键实体都有唯一 ID，并能在页面中追踪上下游关系。

### 4. AI 不是单独功能

AI 在本原型中不是一个孤立聊天框，而是作为序列设计、实验解析、结果推荐和知识问答的交互方式嵌入工作流。

## 技术栈

- `Streamlit`：多页面应用框架
- `Plotly`：交互式图表
- `Pandas`：表格与数据处理
- `NumPy`：曲线模拟与数值计算

## 本地运行

### 安装依赖

```bash
pip install streamlit plotly pandas numpy
```

### 启动项目

```bash
cd abdiscovery-os
streamlit run app.py --server.port 8504
```

启动后访问：

`http://localhost:8504`

## 数据说明

项目当前全部使用 Mock 数据，但数据结构是按真实抗体研发流程组织的，覆盖：

1. 在研项目
2. 抗体序列
3. 细胞株
4. 质粒
5. ELISA 数据
6. SPR 数据
7. 表达数据
8. 干湿闭环迭代数据
9. 靶点与竞品情报
